# Cursor Configuration

Este diretório define regras e planos de uso profissional do Cursor.

Objetivos:
- Garantir testabilidade
- Padronizar explicações de código
- Reduzir dívida técnica
- Aumentar previsibilidade
- Controlar trade-offs conscientemente

Regras (.cursor/rules):
- Aplicadas automaticamente em toda interação

Planos (.cursor/plans):
- Devem ser escolhidos explicitamente antes de gerar código
