# Refactor-First Plan

Objetivo: Tornar código existente testável antes de evoluir.

Passos:
1. Explicar comportamento atual
2. Criar testes de caracterização
3. Refatorar minimamente para testabilidade
4. Validar com testes
5. Só então evoluir funcionalidade

Regras:
- Manter comportamento existente
- Mudanças pequenas e seguras
