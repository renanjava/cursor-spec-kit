# Code Explanation Standard

Sempre que explicar código (gerado ou existente), siga exatamente este formato:

1. **Resumo**
   - Explique em 1–2 frases o propósito do código.

2. **Responsabilidades**
   - Liste as responsabilidades principais.

3. **Fluxo de Execução**
   - Explique passo a passo o que acontece em tempo de execução.

4. **Dependências**
   - Cite dependências internas e externas relevantes.

5. **Efeitos Colaterais**
   - Mutações de estado, I/O, chamadas externas, logs.

6. **Casos de Erro**
   - Erros possíveis e como são tratados.

7. **Pontos de Atenção**
   - Performance, concorrência, segurança, complexidade.

8. **Sugestões**
   - Possíveis melhorias ou refatorações.

Nunca omitir seções.
Se não aplicável, escrever explicitamente: “Não aplicável”.
